About This Extension

Catalog Visibility is a Magento extension that allows store owners to see the reasons that contribute to the salability and visibility of their catalog products. Store owners will be able to easily verify that they have correctly set up their products with this module.

Installation

  1. Back Up Your Data - Any installation can go wrong, so make sure all to back up your code and database before installing any new extension.
  2. Extract Files - Unzip the files from this archive into some location you can get to.
  3. Upload Data - Using a FTP client (or SCP transfer), copy the directories from the unzipped archive into the app directory of your Magento installation. See below for specific file copying instructions.
  4. Refresh Your Cache - Most installation problems can be alleviated by clearing your site and browser cache. Clear cache in Magento, then logout and log back in for best results. Just try it, you'll see!
  5. Log Out and Log Back In - Magento needs to install the new module, and logging out and then back in should cause it to do this.
  6. Troubleshooting - If you don't see any changes, take a look at System -> Configuration -> Advanced and see if MM_Visibility is listed in the modules list. If so, the module is installed, and some caching is probably at fault.

Copying Files

Many FTP clients will copy subdirectories properly when you attempt to copy a higher level directory. If not, you will need to copy folders from the extension directory manually.

  1. Copy adminhtml to %magento%/app/design. If you are using a theme other than default/default, you may want to copy adminhtml/default/default/template into %magento%/app/design/adminhtml/default/%yourtheme%/template
  2. Copy MM into %magento%/app/code/local. You may have to create this directory if it does not exist.
  3. Copy modules/MM_CatalogVisibility.xml into %magento%/app/etc/modules
  4. Copy en_US/MM_CatalogVisibility.csv (and any other language CSV files) into %magento%/app/locale/%language%/
  5. package.xml is used to automate deployments and can be safely ignored.
